User Roles & Views

Every user gets a role-scoped view. State is tracked per user, per task.

New Starter

  • Personal task board (Jira-style)
  • Progress bar & milestone tracker
  • Training status per module
  • Ask Bob contextual help
  • FAQ & quick links
  • 30/60/90 day plan view

Manager

  • View all direct reports' progress
  • Assign or override tasks
  • Personalise 30/60/90 goals
  • Flag overdue items
  • Book 1:1 via calendar plugin
  • Add custom notes per hire

HR / Admin

  • Org-wide onboarding analytics
  • Configure plugins & task templates
  • Manage cohorts & teams
  • Upload policies & documents
  • Set mandatory vs optional tasks
  • Audit log of completions

IT / Support

  • View pending tool provisioning
  • Mark tools as set up
  • Receive auto-generated tickets
  • Track hardware requests

Per-User Task State — How It Works

Like Jira issues, every task has a persistent state record tied to a user. Completion is audited.

Task States

Not Started → In Progress → Blocked → Done → Verified

Due Date Logic

Deadlines are relative offsets from start date (e.g. Day 1, Day 5, Day 30) — not fixed calendar dates

Evidence Capture

Tasks can require proof: certificate upload, manager sign-off, or system auto-verify (e.g. LMS API)

Notifications

Overdue alerts via email or Slack. Manager notified when a mandatory item is blocked or missed

Bulk Assignment

Task templates applied by role, team, location, or cohort. One config, hundreds of hires

Completion Audit

Every status change is logged with timestamp, user, and method (self-reported vs auto-verified)
